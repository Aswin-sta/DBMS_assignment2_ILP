Fitness Tracking Application Database

Aswinkumar TS

Domain Overview

The Fitness Tracking Application domain is highly relevant in today's health-conscious society. This application allows users to monitor and improve their health and fitness levels by tracking physical activities, nutrition, sleep patterns, and overall well-being.

Key Features

  1. User Profiles:
    • Users can create and manage profiles with personal information, including age, weight, height, and fitness goals.
  1. Authentication and Security:
    • The application ensures secure user authentication with password hashing and token-based access.
  1. Workouts and Categories:
    • Users can log different types of workouts, categorized for better organization.
  1. Sleep Logging:
    • Users can track sleep patterns, including duration and quality.
  1. User Goals and Achievements:
    • Users can set fitness goals, and the application helps track progress.

Database Schema

CREATE TABLE user (

id SERIAL PRIMARY KEY,

username VARCHAR(50) NOT NULL UNIQUE,

email VARCHAR(100) NOT NULL UNIQUE,

created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

);

CREATE TABLE role (

id SERIAL PRIMARY KEY,

role_name VARCHAR(50) NOT NULL UNIQUE

);

CREATE TABLE user_role (

user_id INTEGER REFERENCES user(id),

role_id INTEGER REFERENCES role(id),

PRIMARY KEY (user_id, role_id)

);

CREATE TABLE login_history (

id SERIAL PRIMARY KEY,

user_id INTEGER REFERENCES user(id),

login_time TIMESTAMP,

logout_time TIMESTAMP,

device_id VARCHAR(255)

);

CREATE TABLE customer (

id SERIAL PRIMARY KEY,

location VARCHAR(50),

age INT CHECK (age >= 0),

birthdate DATE,

weight DECIMAL(5,2) CHECK (weight >= 0),

height DECIMAL(5,2) CHECK (height >= 0)

);

CREATE TABLE customer_user_mapping (

customer_id INTEGER REFERENCES customer(id),

user_id INTEGER REFERENCES user(id),

PRIMARY KEY (customer_id, user_id)

);

CREATE TABLE authentication_token (

id SERIAL PRIMARY KEY,

user_id INTEGER REFERENCES user(id),

token_hash VARCHAR(255) NOT NULL,

expiration_time TIMESTAMP NOT NULL

);

CREATE TABLE password (

id SERIAL PRIMARY KEY,

user_id INTEGER REFERENCES user(id),

password_hash VARCHAR(255) NOT NULL

);

CREATE TABLE workouts_category (

id SERIAL PRIMARY KEY,

category_name VARCHAR(50) NOT NULL UNIQUE,

image_url VARCHAR(255)

);

CREATE TABLE workout (

id SERIAL PRIMARY KEY,

category_id INTEGER REFERENCES workouts_category(id),

description TEXT,

calories_consumed_per_minute DECIMAL(8,2) CHECK (calories_consumed_per_minute >= 0)

);

CREATE TABLE customer_activity (

id SERIAL PRIMARY KEY,

customer_id INTEGER REFERENCES customer(id),

workout_id INTEGER REFERENCES workout(id),

workout_date TIMESTAMP,

average_heart_rate INTEGER,

distance DECIMAL(8,2),

duration_minutes INTEGER CHECK (duration_minutes >= 0)

);

CREATE TABLE sleep_log (

id SERIAL PRIMARY KEY,

customer_id INTEGER REFERENCES customer(id),

date DATE,

duration_hours DECIMAL(5,2) CHECK (duration_hours >= 0),

quality_id INTEGER REFERENCES sleep_quality(id)

);

CREATE TABLE sleep_quality (

id SERIAL PRIMARY KEY,

quality_name VARCHAR(50) NOT NULL UNIQUE

);

CREATE TABLE daily_goal (

id SERIAL PRIMARY KEY,

steps_goal INT,

calories_burned_goal INT

);

CREATE TABLE user_daily_progress (

id SERIAL PRIMARY KEY,

customer_id INTEGER REFERENCES customer(id),

date DATE,

steps_achieved INT,

calories_burned_achieved INT

);

CREATE TABLE daily_goal_customer_progress (

id SERIAL PRIMARY KEY,

goal_id INT REFERENCES daily_goal(id),

progress_id INT REFERENCES user_daily_progress(id)

);

User Onboarding Journey

1. User Login:

  • The user provides their email and password for login.

2. Credential Verification:

  • The system verifies the provided credentials.

3. Successful Login:

  • If login is successful, the system identifies the user's role (admin or customer).

4. Customer Data Update:

  • If the user is a customer, relevant data is added to the customers table.
  • Data includes location, age, birthdate, weight, and height.

5. Access to Personalized Dashboard:

  • The user gains access to a personalized dashboard.
  • Views fitness goals, achievements, and wellness tracking data.

6. Admin Access:

  • If the user is an admin, access to administrative features is granted.

7. Setting Fitness Goals:

  • Users select personalized daily goals from predefined options, and the app dynamically tracks their progress, providing insights and adaptive suggestions for continuous motivation and improvement.

8. Selecting Workouts:

  • The user chooses workouts from predefined options stored in the workouts table.
  • Details of selected workouts are recorded in the user_workouts table, including metrics like duration and heart rate.

9. Quality Sleep Logging:

  • The user logs sleep patterns, with details stored in the sleep_log table.
  • Information includes sleep duration, date, and quality (Good, Average, or Poor).
